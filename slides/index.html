The past

vue-cli 1

vue-cli 2

Vue-cli 1 and 2 both allowed the user to scaffold an app conveniently, but after the initial setup, they were left on their own. The versions didn't have any breaking changes between them and version 2's major change was the addition of prompts instead of a schema file and a better CLI experience in general.

CLI

All this is made possible through a magical vue-cli-service library, that hides away the mess, yet let's you keep control of everything.

CLI commands

serve

The serve command is a powerful part of the vue-cli-service, allowing you to serve even a single .vue file without any config or overhead of the project setup!

The drawback of vue serve is that it relies on globally installed dependencies which may be inconsistent on different machines. Therefore this is only recommended for rapid prototyping.

build

vue-cli-service build command exposes multiple build targets like app, lib, wc, wc-async
